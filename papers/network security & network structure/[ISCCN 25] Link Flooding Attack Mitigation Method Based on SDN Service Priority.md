## 연구 배경 및 문제

### Link Flooding Attack (LFA)란?
- 전통적 DDoS와 달리 **서비스가 아닌 링크를 공격 대상**으로 삼는 새로운 유형의 DDoS 공격
- 공격자가 대상 서비스로 향하는 핵심 링크를 혼잡시켜 정당한 사용자의 접근을 차단
- 특성: 실제/합법적 IP 사용, 단일 공격 호스트의 대역폭 점유 비율 낮음, 정상 트래픽과 높은 유사성 → **탐지 매우 어려움**

### LFA의 두 가지 형태
| 형태 | 설명 |
|------|------|
| **Coremelt Attack** | 공격자가 악성 호스트 간 상호 데이터 전송으로 대상 링크 혼잡 |
| **Crossfire Attack** | 악성 호스트를 소스, 공개 서비스(디코이)를 목적지로 사용. 3단계로 진행: ① 네트워크 토폴로지 탐지 → ② 대상 링크 선택 → ③ 롤링 공격 (주기적 링크 전환) |

### 기존 방어 방법의 한계
| 접근법 | 한계 |
|--------|------|
| **전통적 DDoS 탐지** (임계값, 규칙, DPI) | LFA의 정상 트래픽 유사성·저대역폭 특성으로 적용 불가 |
| **탐지 패킷 차단/리다이렉트** (traceroute 감지) | 공격 호스트 비율 낮으면 정보 불완전, 정상 사용자도 탐지 도구 사용 불가 |
| **재라우팅 기반 완화 후 탐지** (CFADefense 등) | 대안 경로 부족 시 완화 불가, 악성 호스트 탐지에 다수 라운드 상호작용 필요 |
| **전통적 DDoS 특징 기반 ML** (CyberPulse++) | LFA 특성이 아닌 전통 DDoS 특징 사용 → LFA 탐지에 부적합 |

## SPM 방법 설계

### 핵심 아이디어
- **서비스 우선순위 기반**: 네트워크 전체에 균일한 방어가 아닌, **고가치 대상 서비스에 집중** → 서비스↔링크 대응관계 명확화, 탐지·방어 비용 효과적 감소
- SDN의 글로벌 가시성(포워딩/컨트롤 분리, 커스터마이즈 가능한 제어 서비스) 활용
- LFA 자체의 행동 특성에서 피처 추출 (전통 DDoS 피처 탈피)
- **빠른 완화 + 느린 식별(quick mitigation, slow detection)** 전략

### 위협 모델
- 텔레콤 사업자 또는 데이터센터 네트워크, 모든 라우터가 SDN 스위치
- 방어자는 SDN 컨트롤러 전역 권한 보유, 최단 경로 우선 라우팅
- 소수 링크가 다수 라우팅 경로에 동시 존재 → 높은 중요도
- 공격자는 악성 호스트 그룹을 조종하여 핵심 링크 대상 flooding 공격 수행

### 시스템 아키텍처 (2개 핵심 모듈)

```
[SDN Controller]
  - 핵심 링크 선택 (서비스 기반 중요도 계산)
  - LLDP 기반 링크 지연 계산 (5초 주기)
  - 혼잡 감지 → 공격 탐지·완화 모듈에 위치 정보 전달
         ↓
[Attack Detection & Mitigation Module]
  - 혼잡 스위치에서 flow table 정보 직접 요청 (1초 주기, 5초간)
  - 공격 행동 특징 추출 → ML 모델 판단
  - 악성 호스트 → 소스 스위치에 DROP flow table 전송
```

### 탐지 단계

#### 1. 핵심 링크 선택 (Algorithm 1)
- 각 대상 서비스 $s$에 대해, 네트워크 내 모든 스위치 $v$에서 서비스 스위치 $P(s)$까지의 라우팅 경로 수집
- 각 링크 $e$가 몇 개의 경로에 포함되는지 빈도 계산 → 상위 $k$%(파레토 원칙, 20%) 링크를 핵심 링크로 선정

#### 2. 링크 혼잡 감지
- **LLDP 프로토콜 기반 링크 지연 측정**: 송수신 타임스탬프 추가
  - $link\_delay = (T_1 + T_2 - T_A - T_B) / 2$
  - $T_1, T_2$: 인접 스위치 LLDP 패킷 수신→컨트롤러 보고 지연, $T_A, T_B$: 컨트롤러↔스위치 왕복 지연
- 5초 주기로 혼잡 상태 감지 (편도 전파 지연 100ms 도달 시 혼잡 판정)
- 공격 대역폭이 특정 임계값 초과 시 지연 급증 — 대역폭별 임계값 개별 설정 필요

#### 3. 실험적 발견
- 링크에 3ms 추가 지연 환경에서, 공격 대역폭이 임계 비율 미만일 때 지연 안정 → 초과 시 급증
- 대역폭 증가 시 지연 변곡점이 앞당겨짐

### 완화 단계

#### 1. 공격 특징 추출 (11개 피처)

| 피처 | 정의 | 설계 근거 |
|------|------|---------|
| **목적지 IP 수** | 시간 윈도우 내 동일 소스 IP의 목적지 IP 개수 | 정상 호스트는 적음, 롤링 공격 시 디코이 서비스 빈번 전환 |
| **고빈도 목적지 IP 수** | 고빈도 목적지 IP 개수 | 악성 호스트가 고빈도 목적지에 집중 |
| **고빈도 목적지 IP 비율** | 고빈도 목적지 IP / 전체 목적지 IP | 악성 호스트 패턴 집중도 |
| **현재 바이트 수** | 현재 시점 소스 IP의 전송 바이트 | 혼잡 시 고대역폭 우선 완화 |
| **현재 패킷 수** | 현재 시점 소스 IP의 전송 패킷 | 동일 |
| **패킷당 바이트 수** | 현재 시점 패킷당 평균 바이트 | 공격 호스트 패턴 단일성 |
| **총 바이트 수** | 시간 윈도우 내 총 전송 바이트 | 히스토리컬 트래픽 볼륨 |
| **총 패킷 수** | 시간 윈도우 내 총 전송 패킷 | 동일 |
| **현재 바이트 비율** | 현재 바이트 / 윈도우 내 총 바이트 | 정상 호스트의 돌발적 접근 패턴 vs 공격의 균일 패턴 |
| **현재 패킷 비율** | 현재 패킷 / 윈도우 내 총 패킷 | 동일 |
| **혼잡 지속 시간** | 소스 IP가 혼잡 기간에 출현한 누적 시간 | 오래 출현할수록 악성 가능성 높음 |

- 시간 윈도우: 60초
- 동일 소스 IP의 여러 flow table 레코드를 통합 처리
- 상위 $K$% 트래픽 볼륨 목적지 IP를 고빈도 목적지로 분류

#### 2. ML 모델 선택

6개 모델 비교 평가 결과:

| 모델 | 정확도 | 정밀도 | 재현율 | 오탐률 | 학습 시간 | 테스트 시간 |
|------|--------|--------|--------|--------|---------|----------|
| LR | 82.81% | 83.78% | 96.31% | 18.68% | 0.33s | 5.96ms |
| **RF** | **98.19%** | **98.56%** | **99.09%** | **4.84%** | 7.94s | 217.74ms |
| SVM | 84.77% | 86.89% | 94.47% | 14.37% | 370.29s | 15756.97ms |
| NB | 77.48% | 81.64% | 91.29% | 25.02% | 0.01s | 3.79ms |
| DT | 97.30% | 98.16% | 98.34% | 6.28% | 0.47s | 3.75ms |
| GBT | 96.73% | 97.09% | 98.71% | 9.87% | 10.66s | 28.22ms |

→ **Random Forest 선택**: 최고 정확도(98.19%), 최저 오탐률(4.84%), 학습/테스트 시간 수용 가능

#### 3. 완화 전략: 빠른 완화 + 느린 식별 (Algorithm 2)

```
1. ML 모델로 호스트의 악성 확률 계산
2. 확률 ≥ 신뢰 임계값(CT=0.95) → 의심 호스트로 판정
3. 해당 소스 IP의 과거 의심 횟수(count) 조회
4. 폐기 시간 = 2^(count+1) — 지수적 증가
5. 소스 스위치에 높은 우선순위 DROP flow table 전송
6. count+1 ≥ 식별 임계값(MT=5) → 최종 악성 호스트로 확정, 완전 차단
```

**장점**:
- 초기 오판으로 인한 정상 호스트 영향 최소화 (짧은 폐기 시간부터 시작)
- 재라우팅 없이 즉시 혼잡 완화
- 다수 라운드 상호작용 불필요

## 평가

### 실험 환경
- **토폴로지**: 미국 교육 백본 네트워크 Abilene (11 노드, 15 링크), Mininet 시뮬레이션
- 링크 대역폭: 20Mbps, 편도 전파 지연: 3ms, 혼잡 기준: 편도 지연 100ms
- 각 스위치 하위에 3개 호스트 (악성 M, 정상 N, 디코이 서비스 S)
- 핵심 링크: L1, L2, L3 (파레토 원칙 k=20% 적용)

### 트래픽 설정
- **정상 트래픽**: CIC-IDS2017 Monday 데이터 (모두 benign) Tcpreplay, 호스트당 약 10,000 패킷/분
- **공격 트래픽**: 공격 소스 M당 30개 악성 IP 시뮬레이션, 라운드당 총 90개 악성 호스트 행동, 전체 1,200개 악성 호스트, 공격 주기 15초, 정상 트래픽 시작 5분 후 공격 개시, 10분 지속

### 임계값 최적화
| 파라미터 | 최적값 | 근거 |
|---------|--------|------|
| 신뢰 임계값 ($C_T$) | 0.95 | <0.95: 높은 오탐률, >0.95: 재현율 급감 |
| 식별 임계값 ($M_T$) | 5 | 오탐률과 재현율 간 최적 균형 |

### 비교 방법과의 성능 비교

| 비교 축 | SPM | CyberPulse++ | CFADefense |
|--------|-----|-------------|------------|
| **실시간 탐지 속도** | 5분 시 재현율 94.68% | 5분 시 88.35% | 25라운드 이후에야 안정화 |
| **최종 성능 (10분)** | 재현율 97.52%, 오탐률 1.36% | 재현율 94.58%, 오탐률 3.40% | 정밀도 91.56%, 재현율 46.57%, 오탐률 12.91% |
| **악성 호스트 수 증가 시** | 재현율 감소폭 작음, 안정적 | 재현율 감소폭 큼 | 소수일 때만 효과 |
| **공격 주기 변화 시** | 재현율 >90%, 오탐률 <1.5% 항상 유지 | SPM 대비 열세 | 주기 증가 시 오탐률 급증 |

## 핵심 기여 요약

1. **서비스 우선순위 기반 핵심 링크 선택**: 서비스↔링크 대응관계 구축으로 방어 범위를 좁혀 비용 효과적 모니터링
2. **LFA 특화 행동 피처 추출**: 전통 DDoS 피처 대신 SDN flow table에서 고빈도 목적지 집중도, 혼잡 기간 출현 패턴 등 LFA 고유 행동 특성 11개 추출
3. **빠른 완화 + 느린 식별 전략**: 지수적 폐기 시간 증가 + 이중 임계값(신뢰/식별)으로 정상 호스트 영향 최소화하면서 즉시 혼잡 완화

## 한계 및 향후 과제

- Mininet 시뮬레이션 환경(20Mbps, 11 노드)으로 대규모 실제 네트워크 검증 부족
- 멀티 컨트롤러 환경 미지원
- SDN 스위치 부분 배포 시나리오 미고려
- 단일 위협 모델(Crossfire 형태)만 평가, Coremelt 등 다른 LFA 변종 미검증
- 공격자가 피처 추출 윈도우(60초) 이내에 행동 패턴을 변경하는 적응형 공격에 대한 내성 미평가
